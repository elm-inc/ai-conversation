# Design

設計・実装方針のドキュメント。ADR とは異なり「これから何をどう作るか」「実装にあたっての方針」を扱う。

## 想定するファイル

| ファイル | 内容 |
|---|---|
| `implementation-plan.md` | モジュール構成、アーキテクチャパターン、実装順序 |
| `<機能名>-spec.md` | 個別機能の詳細設計 |
| `api-spec.md` | API 設計 (必要に応じて) |

## ADR との違い

- **ADR**: 過去に下した**決定の理由**を残す (immutable, 履歴として残る)
- **design**: 現在の**設計・方針**を表現する (随時更新)

`docs/design/` のドキュメントが古くなったら **直接書き換える**。歴史的経緯を残したい部分は ADR を新規作成。
